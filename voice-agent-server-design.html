1. 全体像

VoiceAgent サーバーは、クライアントと外部AI / 音声 / 楽曲サービスの間に置かれる中継サーバーです。CORS、認証、セッション、音声整形、楽曲ライブラリ、ドライブDJ用の地域情報をサーバー側で統合します。

クライアント中継

Web UI と Android ネイティブAPIを同一アプリで提供し、認証方式をルートごとに切り替えます。

AI / 音声中継

Whisper、OpenClaw、TTS へのリクエストをサーバー側で組み立て、結果をクライアントへ返します。

DJ / 音楽管理

地域情報、天気、名所、楽曲一覧、音量補正をまとめてドライブDJへ提供します。

3. 認証

Web UI とネイティブAPIで認証方式が分かれます。静的Web UIと通常 /api/ ルートは Basic 認証、/api/native/ ルートはネイティブAPIキー認証を使います。

対象 認証方式 設定
Web UI / /api/* Basic認証。ユーザー名とパスワードが両方空なら無効。 BASIC_AUTH_USER, BASIC_AUTH_PASSWORD
/api/native/* ネイティブAPIキー。Authorization, X-VoiceAgent-Key, X-API-Key を受け付ける。 VOICEAGENT_NATIVE_API_KEY
上流API none, raw, bearer, x-api-key に対応。 *_AUTH_MODE, *_API_KEY

4. APIエンドポイント

ほとんどのAPIは Web向け /api/... と Android向け /api/native/... の両方を持ちます。機能は基本的に同じで、認証方式だけが異なります。

メソッド パス 用途
GET/api/config, /api/native/config機能有効状態、既定ユーザー、DJ既定地域を返す。
POST/api/transcribe, /api/native/transcribe音声ファイルをWhisperへ中継し、文字起こし結果を返す。
POST/api/chat/stream, /api/native/chat/streamOpenClawへストリーミングチャットを中継する。
POST/api/drive-dj/stream, /api/native/drive-dj/streamDJ紹介文生成用のLLMストリームを中継する。
POST/api/speech, /api/native/speechTTSへ中継し、音声バイナリを返す。
POST/api/compact, /api/native/compactOpenClawセッションのコンパクト化を要求する。
GET/api/drive-dj/location, /api/native/drive-dj/locationDJ用の地域、天気、名所情報を返す。
GET/api/music/playlist, /api/native/music/playlist楽曲一覧を返す。必要に応じて再スキャンする。
GET / HEAD/api/music/tracks/{track_id}, /api/native/music/tracks/{track_id}楽曲ファイルを配信する。
GET/api/music/volume-overrides, /api/native/music/volume-overrides楽曲ごとの音量補正値を返す。
POST/api/music/volume-overrides, /api/native/music/volume-overrides楽曲ごとの音量補正値を保存する。
POST/api/client-log, /api/native/client-logクライアントイベントログをサーバーログに記録する。
GET / HEAD/, /{path}Web UI静的ファイルを返す。

5. 会話ストリーム

通常会話はクライアントから受け取ったメッセージに地域コンテキストを追加し、OpenClawのChat Completions互換APIへ stream: true で中継します。

項目 仕様
上流OpenClaw通信httpx.AsyncClient(timeout=None)
クライアントへの形式text/event-stream
keep-alive15秒ごと
セッションuser と _meta.sessionKey を上流へ渡す。
セッションリセットresetSession が真なら _meta.resetSession を付加。
コンパクトAPI上流へ最大180秒でPOST。

6. 音声認識とTTS

Whisper中継

アップロードされた音声ファイルをそのまま上流の文字起こしAPIへ multipart/form-data で送信します。

入力UploadFile
上流タイムアウト120秒
言語既定は日本語。環境変数で変更可能。

TTS中継

読み上げ前にテキストをサニタイズし、TTS APIへ送信します。音声以外のレスポンスはエラーとして扱います。

入力{"text": "..."}
上流タイムアウト120秒
出力音声バイナリ

TTSサニタイズ

TTSへ渡す前に、URL、メールアドレス、長い英数字トークン、Markdownリンク、コードフェンス、括弧内の演出指示などを除去または読点相当へ置換します。空文字になった場合は400エラーを返します。

7. ドライブDJ

ドライブDJでは、地域情報、天気、ランドマーク、楽曲リストを組み合わせて紹介文を作ります。Androidアプリ側では3曲1セットの紹介と連続再生に使われます。

要素 仕様
位置情報任意の検索APIを使用。未設定、失敗、結果なしの場合は既定地域。
天気補助スクリプトをサブプロセスで実行。既定タイムアウトは8秒。
天気キャッシュ既定で1800秒。
ランドマークJSONデータセットを読み込み、地域名の正規化キーで検索。
DJ用LLM専用URLがあれば使用。未設定ならOpenClawへフォールバック。
プロンプトsettings/prompts/ のテンプレートを使用。

8. 楽曲配信

音楽ライブラリは読み取り専用マウントを前提に、サーバーが対応拡張子を再帰スキャンしてプレイリストを作成します。タグが読める場合はアーティスト、アルバム、タイトルを利用します。

項目 仕様
対応拡張子.mp3, .m4a, .aac, .wav, .flac, .ogg, .opus, .webm
曲IDファイル相対パス由来のダイジェスト。
タグ読み取りmutagen でApple Music系タグなどを取得。
メモリキャッシュ既定600秒。
プレイリスト再スキャン起動時と定期バックグラウンド更新。既定3600秒。
音量補正補正値は -18 から +12 に丸め、0は保存しない。
楽曲配信FileResponse で音声MIMEを付けて返す。

9. Web UI / PWA

voiceagent にはブラウザ向け UI も含まれます。ブラウザでは録音、テキスト会話、画像添付、音声再生、ドライブDJ、楽曲再生を実装しています。

Web UI

  • React UMD + htm によるビルドレス構成
  • ブラウザ録音と音声レベル表示
  • SSEによる会話ストリーム表示
  • TTS再生キュー
  • 画像添付とリサイズ

PWA

  • Web manifest
  • Service Worker
  • アプリアイコン
  • 静的アセット配信
Androidネイティブアプリの主要な利用経路は /api/native だが、Web UI は同じサーバー上で保守用・ブラウザ利用用として動作する。

10. 設定

設定は主に環境変数で注入します。以下は設計上の分類であり、具体的なURL、トークン、ホストパスはドキュメントに記載しません。

分類 主な環境変数 用途
アプリ APP_TITLE UI表示名。
Web認証 BASIC_AUTH_USER, BASIC_AUTH_PASSWORD Web UI と通常APIのBasic認証。
ネイティブAPI認証 VOICEAGENT_NATIVE_API_KEY AndroidアプリからのAPIアクセス制御。
Whisper WHISPER_URL, WHISPER_API_KEY, WHISPER_AUTH_MODE, WHISPER_MODEL 音声文字起こし。
OpenClaw OPENCLAW_URL, OPENCLAW_COMPACT_URL, OPENCLAW_API_KEY, OPENCLAW_MODEL 通常会話とセッション管理。
ドライブDJ LLM DRIVE_DJ_LLM_URL, DRIVE_DJ_LLM_API_KEY, DRIVE_DJ_LLM_MODEL DJ紹介文専用LLM。未設定ならOpenClawへフォールバック。
位置 / 天気 DRIVE_DJ_LOCATION_DEFAULT, DRIVE_DJ_LOCATION_ES_URL, DRIVE_DJ_WEATHER_ENABLED, DRIVE_DJ_WEATHER_TIMEOUT DJ用地域情報。
TTS TTS_URL, TTS_API_KEY, TTS_MODEL, TTS_VOICE, TTS_RESPONSE_FORMAT, TTS_SPEED 音声合成。
音楽 MUSIC_LIBRARY_PATH, MUSIC_CACHE_SECONDS, MUSIC_PLAYLIST_REFRESH_SECONDS, MUSIC_MOUNT_WAIT_SECONDS 楽曲ライブラリとキャッシュ制御。

11. 実行環境

コンテナ実行を基本とし、起動時に音楽ライブラリのマウント待機を行ってから Uvicorn で FastAPI を起動します。

主要依存

依存 用途
fastapiAPIサーバー。
uvicorn[standard]ASGI実行環境。
httpx上流APIへの非同期HTTP通信。
python-multipart音声ファイルアップロード。
mutagen音楽タグ解析。

汎用コマンド例

# 環境変数を設定したうえで起動
docker compose up -d --build

# ログ確認
docker compose logs --tail=100 voiceagent

12. 注意点と改善候補

注意点

  • OpenClawストリームは上流HTTPタイムアウトなしのため、クライアント側タイムアウトとの整合が重要。
  • Basic認証が未設定だとWeb UI保護は無効になる。
  • ネイティブAPIキー未設定時は /api/native が利用できない。
  • 楽曲ライブラリの初回スキャンはファイル数やマウント状況に依存する。
  • TTSサニタイズは安全側に寄せており、読み上げ内容が短くなる場合がある。

改善候補

  • 設定値のヘルスチェックAPI追加。
  • プロンプトテンプレートの管理画面化。
  • 楽曲キャッシュとスキャン状況の可視化。
  • 上流APIごとのタイムアウト設定を環境変数化。
  • 構造化ログとリクエストIDの導入。
  • ネイティブAPIキーのローテーション対応。
この設計書では、実URL、IPアドレス、ユーザー名、ホスト上の絶対パス、APIキーなどの環境固有情報は記載しません。運用時は環境変数またはデプロイ手順書で管理します。
