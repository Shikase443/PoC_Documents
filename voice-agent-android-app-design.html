1. 全体像

アプリは UI とフォアグラウンドサービスを分離し、録音、通信、再生、DJモードはサービス側で集中的に管理します。外部AIや音声生成へは Android から直接接続せず、VoiceAgent サーバーのネイティブAPIを経由します。

UI

ユーザー操作、設定保存、状態表示、チャット履歴、画像添付を担当します。

サービス

録音、OpenClaw待機、TTS再生、DJモード、通知、WakeLockを担当します。

サーバー連携

Whisper、OpenClaw、TTS、楽曲配信は VoiceAgent API 経由で利用します。

2. 主要コンポーネント

ファイル 責務
MainActivity.kt Compose UI、権限要求、設定保存、サービス操作、ブロードキャスト受信。
VoiceAgentForegroundService.kt 録音、音声会話、TTS、DJモード、通知、状態配信の中核。
VoiceAgentApiClient.kt VoiceAgent API との HTTP 通信、SSE応答の解析。
VoiceRecorder.kt MediaRecorder による M4A/AAC 録音、Bluetoothマイク経路制御。
VoicePlayer.kt MediaPlayer による TTS / 楽曲再生、唯ちゃん音声の音量底上げ。
SpeechEnvelope.kt TTS音声の波形解析と口パク表示用レベル算出。
VoiceAgentSettings.kt デフォルトURL、SharedPreferencesキー、セッションキー生成。
HeadsetButtonReceiver.kt メディアボタン入力をサービスへ転送。

3. ライフサイクル

イベント 動作
アプリ起動 権限確認後、ACTION_START_SERVICE でサービスを起動。
サービス起動 通知チャンネル、MediaSession、音声経路監視を初期化。
アプリ終了 isFinishing の場合にサービスを停止。
タスク削除 再生を止め、stopSelf() でサービスを終了。

4. 音声会話

録音前に通信確認を行い、通信できない場合は録音へ進みません。録音結果は Whisper へ送信し、テキスト化後に OpenClaw のストリーム応答を受け取り、文単位のチャンクで TTS 再生します。

録音停止条件

条件 値
監視間隔120ms
最低録音時間1.2秒
音声あり後の無音停止1.6秒
音声なし停止8秒
最大録音時間90秒
音声判定しきい値MediaRecorder.maxAmplitude() >= 700

音声コマンド

発話例 動作
音楽をかけて / 音楽かけて / 音楽を再生してドライブDJモードを開始
音楽を停止して / 音楽を止めて / 音楽をとめて再生停止

5. ドライブDJモード

ドライブDJモードは、地域紹介と3曲紹介を1セットとして生成し、その後に3曲を連続再生します。地域紹介は1回だけ行い、楽曲紹介はシンプルな定型に寄せます。

項目 仕様
1セット曲数3曲
紹介文地域紹介1回 + シンプルな3曲紹介
同曲回避直近15曲を避ける
曲間約4秒
セット間約1.5秒
開始固定音声「ゆいちゃんのお出かけラジオの時間です。」
日時アプリ側の現在日時を注入。GPSレスポンスの日時は使わない。
日時は挨拶や季節感、時間帯判断には使いますが、日時、日付、曜日、時刻そのものは発話しない方針です。

6. API連携

Android アプリは設定画面で指定した <VoiceAgent API Base URL> を基点として、VoiceAgent サーバーへ通信します。

メソッド パス 用途
GETconfig通信確認、設定取得
POSTtranscribe録音ファイルをWhisperへ送信
POSTchat/stream通常会話のOpenClawストリーム
POSTdrive-dj/streamDJ紹介文生成
GETdrive-dj/location地域情報、天気、ランドマーク取得
GETmusic/playlist?refresh=true楽曲一覧取得
GETmusic/tracks/{id}楽曲ファイル取得
GETmusic/volume-overrides楽曲音量補正取得
POSTmusic/volume-overrides楽曲音量補正保存
POSTspeechTTS音声生成

タイムアウト

区分 値
接続タイムアウト20秒
共通読み取りタイムアウト300秒
楽曲ダウンロード読み取りタイムアウト300秒

7. 録音と再生

録音

方式MediaRecorder
形式MPEG-4 / AAC
サンプリング16kHz
ビットレート128kbps
保存先cacheDir/recordings

再生

方式MediaPlayer
TTSCONTENT_TYPE_SPEECH
楽曲CONTENT_TYPE_MUSIC
TTS補正LoudnessEnhancer
口パクSpeechEnvelope

Bluetoothマイクが利用できる場合は、通話モードとBluetooth SCO / BLE headsetを試行します。録音前にマイク経路をウォームアップし、「ご用件をどうぞ」の後の録音開始を速くしています。

8. キャッシュ

種別 保存先 目的 再起動後
固定TTS音声 filesDir/speech_cache 通信不可、待機、録音開始などの定型文 有効
生成TTS音声 cacheDir/speech 会話応答、DJ紹介文の一時再生 OS判断で削除される可能性あり
楽曲ファイル cacheDir/music ダウンロード後の再生 OS判断で削除される可能性あり
録音ファイル cacheDir/recordings Whisper送信用 一時ファイル扱い
固定TTS音声は初回オンライン生成が必要です。キャッシュがない通信不可状態では、音声通知できずUIステータスのみになります。

9. 状態管理

サービスは ACTION_STATUS ブロードキャストでUIへ状態を伝えます。UIはこの状態をもとに、録音中、処理中、DJ中、再生中、現在曲、ログを更新します。

Extra 意味
EXTRA_STATUS現在状態
EXTRA_TRANSCRIPTユーザー発話またはDJ曲リスト
EXTRA_ASSISTANTアシスタント応答またはDJ紹介文
EXTRA_RECORDING録音中フラグ
EXTRA_BUSY処理中フラグ
EXTRA_DRIVE_DJDJモード中フラグ
EXTRA_SPEAKING音声再生中フラグ
EXTRA_SPEECH_LEVEL口パク表示用レベル
EXTRA_MUSIC_ARTIST / TITLE再生中楽曲情報

10. 権限と注意点

権限 用途
INTERNETVoiceAgent API通信
ACCESS_NETWORK_STATE通信状態確認
RECORD_AUDIO音声録音
MODIFY_AUDIO_SETTINGSBluetoothマイク経路制御
WAKE_LOCK録音、会話、再生中のスリープ抑制
FOREGROUND_SERVICE_MICROPHONEフォアグラウンド録音
FOREGROUND_SERVICE_MEDIA_PLAYBACKフォアグラウンド音楽再生
POST_NOTIFICATIONS通知表示
BLUETOOTH_CONNECTBluetoothマイク利用
APIキーは SharedPreferences に保存されます。暗号化保存ではありません。また、usesCleartextTraffic=true のためHTTP接続も許可されています。
